Cómo hacer un archivo Readme

En la pagina de github clicar en CODE y nos lleva al master de nuestro proyecto

En la parte final de la lista del carpetas hay un boton verde ADD AND README, sirve para agregar la descripcion del proyecto: Sirve para hacer una comparacion de lo que tenemos en git y en github

Despueés de clicar nos lleva a otra pagina de nuestro github

Ya aparece su nombre por defecto (README.md)

Si ya tenemos un texto anterior introducido en el comentario podemos modificarlo

Si clicamos sobre preview podemos ver su apariencia

bajamos en la misma pagina y eoncontramos que podemos el archivo haciendo un nuevo commit

Llenamos poniendo: Creando Readme.md

Click en el boton commit new file y nos crea la lista: Creat Readme.md, junto a las otras carpetas

este nuevo archivo no se ha creado dentro de VScode

De github a git

podemos crear archivos directamente en github y tenemos que sincronizar con git (de nuve a pc)

Para esto utilizaremos: el metodo 'pull'

como sabemos el 'git push' sirve para empujar nuestros archivos locales a github, pero el metodo PULL hace exactamente lo contrario, osea compara los archivos que tenemos en github y lo trae el archivo de github a nuestros archivos locales

Con FETCH, si hemos modificado nuestro git y tambien modificado el contenido en github, éste lo modifica en ambos sentidos dejandolos igualitos

TAGS

Para llegar a Tag, clicar en CODE, luego clicar en RELEASES

alli nos muestra: There aren’t any releases here. Los releases nos permiten CREAR versiones de nuestro proyecto y descargar facilmente (No clicamos ningun boton)

en el terminal hacemos: 'git tag version1.0 -m "version 1.0"' (el 1ro es el nombre de la version y la 2da es una descripcion) ) enter

para ver este tag (la nueva version), se utiliza: 'git log --oneline'
